Forcing

-

The way parameter values are computed is flexible and can be configured at setup to allow exogenous forcing by variables describing weather (\({\cal W}(t)\)), vector control coverage (\({\cal V}(t)\)), and variables describing resource availability. Library elements in exDE thus encode model families, or (depending on whether you lump or split model taxonomies) perhaps super-families.

The following sections describe how the code was designed to modularize forcing.

Code

Functions that compute derivatives in exDE should be easy to read. The functions that compute derivatives should look like the math on a page. For example, one version of the Ross-Macdonald model looks like this:

\[\begin{eqnarray} \frac{\textstyle{dM}}{\textstyle{dt}} &=& \Lambda - \Omega \cdot M \\ \frac{\textstyle{dY}}{\textstyle{dt}} &=& fq\kappa (M - Y) - \Omega \cdot Y \\ \frac{\textstyle{dZ}}{\textstyle{dt}} &=& \Upsilon \cdot f_{\tau^{-1}} q_{\tau^{-1}}\kappa_{\tau^{-1}} (M_{\tau^{-1}} - Y_{\tau^{-1}}) - \Omega \cdot Z \\ \end{eqnarray}\]

To encode it, the terms and parameters in the model use the same names: terms are passed (e.g., Lambda and kappa); and parameters are computed and attached by name to MYZpar$now (e.g. MYZpar$now$f and MYZpar$now$Omega) so that the code is easy to read:

with(MYZpar$now,{
  dMdt <- Lambda - Omega %*% M
  dYdt <- f*q*kappa*(M - Y) - Omega %*% Y
  dZdt <- Upsilon %*% (fqkappa_eip * (M_eip - Y_eip)) - Omega %*% Z
})

Note that the term fqkappa_eip is \(f_{\tau^{-1}} q_{\tau^{-1}}\kappa_{\tau^{-1}}\), computed by internalizing the lag term.

Effect Sizes

-

To implement forcing by exogenous parameters, we have developed a generalized way of computing the parameter values in two stages:

  • baseline is a construct to hold the values of parameters before modification by modes of vector control that have independent effect sizes:

    • static - for example, in the Ross-Macdonald model with static parameter values, a parameter can be set to its pre-modification baseline at the start of each step: e.g., \(f = f_0.\)

    • forced - In the Ross-Macdonald model, it is possible to set a changing baseline value at the start of each step: e.g., \(f = F_f(t, {\cal W}(t)).\)

    • forced - In the generalized Ross-Macdonald model, it is possible to set a changing baseline value at the start of each step that is also modified by resource availability: e.g., \(f = F_f(t, {\cal W}(t), B(t)).\) This is called an integrative approach because \(B(t)\) can be used to model the effect sizes of ITNs on blood feeding.

  • now is a construct to hold the values of parameters that are used to compute derivatives in dLdt and dMYZdt

  • EffectSize modifies the values of now

    • The function EffectSize calls functions that execute various modes of vector control. Each one of these must compute an effect size, \(\epsilon\), a multiplicative modifier (i.e. \(\epsilon =1\) has no effect).

    • To avoid conflicts or inadvertent inconsistencies, the values of baseline are not modified further. Instead, the effect sizes modify the values of now

Pseudo-code

  • compute baseline

  • clone: baseline \(\rightarrow\) now

  • effect sizes:

    • use baseline to compute EfSz

    • modify now using EfSz

  • Use now in dMYZdt or dLdt

Setup

-

We adopt the convention of setting up models by configuring a function that calls the baseline parameter:

baseline$x = F_x(t, pars)

The function F_x is an S3 class:

F_x = function(t, pars){
  UseMethod("F_x", pars$Xmod$Fx_pars) 
}

A default option assigns a static value for each parameter:

F_x.0 = function(t, pars){
  pars$Xmod$Fx_pars$x0 
}

which gets setup with a default value for x0 that can be overwritten:

setup_Fx_default = function(pars, x0 = 1){
  Fx_pars = list() 
  class(Fx_pars) <- "0"
  Fx_pars$x0 =  x0
  pars$Xmod$Fx_pars = Fx_pars
  return(pars)
}

The model library expands by adding new functions that force parameters

Alternatives models are added by adding the function:

F_x.alt = function(t, pars){with(pars$Xmod$Fx_pars,{
  (a*t)^b 
})}

And writing code to set it up:

setup_Fx_alt = function(pars, a=1, b=2){
  Fx_pars = list() 
  class(Fx_pars) <- "alt"
  Fx_pars$a = a 
  Fx_pars$b = b 
  pars$Xmod$Fx_pars = Fx_pars
  return(pars)
}

Calling setup_Fx_alt(pars) overwrites a previous version of pars$Xmod$Fx_pars and thus created a new model, derived from another one.

example

NOTE The following example follows the logic for the parameter \(f\), but the same protocols are used for other parameters in the models \(\cal MYZ\) and \(\cal L\)

In the translation to code, the parameter \(f\) is stored as MZpar$now$f and flexibility is implemented through dispatching on the function that computes the baseline F_f:

MYZpar$baseline$f = MYZpar$now$f = F_f(t, pars)

Modularity is implemented by using MYZpar$baseline in functions that compute independent effect sizes for vector control, but using those effect sizes to modify MYZpar$now

Models with the same state space but different ways of setting parameters can thus be encoded through dispatching on F_f

For example, if the baseline parameters were static, then we would use the function:

F_f.0 = function(t, pars){
  MYZpars$f0
}

but we might also choose to let \(f\) be forced as a sigmoidal function of air temperature:

F_f.temp1 = function(t, pars){
  with(pars, 
       f_max*a_ft*Temperature^2/(b_ft+a_ft*Temperature^2) 
}

EIP

-

The EIP can be forced by temperature. In exDE, we have adopted the convention that algorithms should compute and store the EIP for mosquitoes at the moment they become infectious as MYZpar$now$eip

One to define the EIP for P. falciparum infecting mosquitoes is at the moment they become infected, \(\tau(t)\) the mosquito would become infectious at time \[ t+\tau(t) \models Y(t) \left[\rightarrow Z\left(t+\tau\left(t\right)\right) \right].\]

In DDEs, we need to know the EIP for mosquitoes when they become infectious. In Spatial Dynamics of Malaria Transmission [1], we called this quantity \(\tau^{-1}(t)\) a mosquito that became infectious at time \(t\) was infected at time \(t-\tau^{-1}(t)\):

\[ t-\tau^{-1}(t) \models Z(t)\; \left[\leftarrow Y\left(t-\tau^{-1}\left(t\right)\right)\right].\]

Notes

We let \(\tau(t)\) denote the EIP for a cohort of mosquitoes that becomes infected at time \(t\); such that they will become infectious at time \(t+\tau(t)\).
